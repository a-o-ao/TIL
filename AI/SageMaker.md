# Amazon SageMaker

## ■ Amazon SageMaker とは
機械学習モデルの**開発・学習・デプロイ・監視**までを一気通貫で実行できる  
AWSのマネージド機械学習プラットフォーム。

---

## ■ デプロイ（推論）パターン（4種類）

Amazon SageMakerで開発したモデルのデプロイ方法は以下の4つ。

| 推論方式 | 概要 | 主なユースケース |
|---|---|---|
| リアルタイム推論 | 低レイテンシで即時応答 | Web API、レコメンド |
| バッチ推論 | 大量データをまとめて処理 | 定期分析、夜間処理 |
| 非同期推論 | 処理完了を待たずに実行 | 医療画像解析など時間のかかる処理 |
| サーバーレス推論 | インフラ管理不要・自動スケール | リクエスト量が不定なサービス |

---

## ■ モデル運用・監視（MLOps）

| 機能 | 説明 |
|---|---|
| SageMaker Model Monitor | 推論リクエストや結果を継続的に監視 |
| SageMaker Model Dashboard | デプロイ済みモデルの一元管理・可視化 |
| SageMaker Model Cards | モデルの重要情報を文書化（PDF出力可） |
| SageMaker Model Registry | 本番稼働用モデルのカタログ化・バージョン管理 |

---

## ■ CI/CD・実験管理

| 機能 | 説明 |
|---|---|
| SageMaker Pipelines | 機械学習ワークロード向けCI/CD |
| SageMaker Experiments | 学習実験の結果を管理・比較 |
| 自動モデルチューニング | ハイパーパラメータ探索を自動化 |

---

## ■ モデル開発・学習

| 機能 | 説明 |
|---|---|
| SageMaker Training | 指定した計算リソースで学習を実行し履歴を自動記録 |
| SageMaker Debugger | 学習プロセスの監視・デバッグ・最適化 |
| SageMaker JumpStart | 事前学習済みモデルを即利用（生成AIモデル含む） |
| SageMaker Canvas | ノーコードでモデル構築が可能 |

---

## ■ データ準備・特徴量管理

| 機能 | 説明 |
|---|---|
| SageMaker Data Wrangler | ・特徴量エンジニアリング専用ツール<br>・ノーコード / ローコード<br>・前処理の可視化・再利用 |
| SageMaker Processing | データ前処理の並列実行 |
| SageMaker Feature Store | 特徴量を一元管理する専用リポジトリ<br>学習用・推論用で同じ特徴量を使える |
| SageMaker Ground Truth | データのラベル付けを効率化 |
| SageMaker Clarify | データやモデルのバイアス評価 |

※特徴量エンジニアリングとは<br>
機械学習モデルが学習しやすい形に、元データを加工・変換すること

---

## ■ 開発環境

| 機能 | 説明 |
|---|---|
| SageMaker Studio | データ準備〜学習〜デプロイまで対応する統合IDE |

---

## ■ まとめ（要点）
- SageMakerは **MLライフサイクル全体をカバー**
- 推論方式は **4パターン** から選択可能
- CI/CD・監視・文書化など **MLOps機能が充実**
- エンジニア向け〜ノーコード利用まで幅広く対応
